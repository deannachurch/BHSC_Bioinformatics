File formats

Defined file formats used by the bioinformatics community. If you are using new data or trying to use new viewers, it is important to note what formats are supported. Note: while many of these formats have specified usage, there can many variants of these files in circulation and these variants may cause some readers to break.

Note: Formats that are 0-based start counting the first base as 0, while formats that are 1-based start counting the first base at 1. It is important to know whether you are using a 0-based or 1-based system!

SAM/BAM(http://samtools.sourceforge.net/SAM1.pdf): SAM is a tab-delimited file format for expressing sequence alignment and mapping. BAM is the binary form of SAM. (1-based)

BED (http://genome.ucsc.edu/FAQ/FAQformat.html#format1): A generic format typically used to define genomic intervals and describe how these intervals should be rendered in the UCSC browser. (0-based)

bigBed (http://genome.ucsc.edu/FAQ/FAQformat.html#format1.5): a binary transformation of BED that allows for larger datasets to be transferred and displayed more quickly. (0-based)

GFF (http://www.sanger.ac.uk/resources/software/gff/): a format that was defined for describing gene annotation. (1-based)

GTF (http://mblab.wustl.edu/GTF2.html): a refinement of GFF that tries to tighten the specification. (1-based)

GFF3 (http://www.sequenceontology.org/gff3.shtml): another attemp to extend GFF and harden the definition. (1-based)

GVF (http://www.sequenceontology.org/resources/gvf.html): A GFF3 like format that is specific for describing variation. (1-based)

VCF (http://www.1000genomes.org/node/101): Variant call format, developed by the 1000 genomes project to handle both variant defintion and genotype information. (1-based, but uses 0 or N+1 to denote the telomere)

HGVS (http://www.hgvs.org/mutnomen): A reporting format for the description of sequence variants.

MAF (http://genome.ucsc.edu/FAQ/FAQformat.html#format5): Multiple alignment format. (0-based)

WIG (http://genome.ucsc.edu/goldenPath/help/wiggle.html): format for continuous data (like gc percent). (1-based)

FASTQ (http://maq.sourceforge.net/fastq.shtml): A format used for storing the bases and quality scores for a sequence read; used by many tools for analyzing next generation sequence.

FASTA (http://www.ncbi.nlm.nih.gov/BLAST/blastcgihelp.shtml): common format for storing sequence data that is used by many tools.

Galaxy has tools that help you convert one format to another.
